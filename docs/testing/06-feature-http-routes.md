# 06 Feature（HTTP/認証/認可/テナント境界）

## 1. Featureで保証すること
- 認証が必要なルートは未ログインで弾かれる
- ロール/権限で操作が許可/拒否される
- テナントAのユーザーがテナントBのリソースにアクセスできない
- 重要操作が監査ログに残る（代表ケース）

## 2. 代表フロー（おすすめ3本）
1) Ticket作成（Agent）
2) アサイン + 状態変更（Manager/Agent）
3) テナント境界（Agentが他テナントに触れない）

## 3. 403/404の方針（重要）
- テナント境界は通常 404（存在を隠す）を推奨
- 方針を固定し、テストも統一する
